## Visão Geral

O sistema de charts do Blip DS segue um padrão de **composição declarativa**: cada gráfico é montado combinando um componente de container, um componente de tipo de gráfico e componentes filhos de configuração. Nenhum filho renderiza algo visível — eles apenas passam configuração ao pai via `querySelector`.

```
bds-chart-container          ← wrapper responsivo
  └── bds-chart-bar          ← tipo do gráfico
        ├── bds-bar          ← define uma série de barras
        ├── bds-x-axis       ← configura o eixo X
        ├── bds-y-axis       ← configura o eixo Y
        ├── bds-chart-grid   ← configura o grid
        ├── bds-chart-legend ← configura a legenda
        └── bds-chart-tooltip← configura o tooltip
```

## bds-chart-heatmap — Mapa de Calor

Ideal para visualizar densidade ou frequência em uma grade bidimensional (ex: hora × dia da semana).

### Propriedades

<table>
  <thead><tr><th>Propriedade</th><th>Tipo</th><th>Padrão</th><th>Descrição</th></tr></thead>
  <tbody>
    <tr><td>`data`</td><td>`ChartDatum[] | string`</td><td>`[]`</td><td>Array de dados. Cada item precisa de três chaves: para X, para Y e para o valor</td></tr>
    <tr><td>`x-key`</td><td>`string`</td><td>`'x'`</td><td>Chave do campo que define o eixo X (ex: dia da semana)</td></tr>
    <tr><td>`y-key`</td><td>`string`</td><td>`'y'`</td><td>Chave do campo que define o eixo Y (ex: hora do dia)</td></tr>
    <tr><td>`value-key`</td><td>`string`</td><td>`'value'`</td><td>Chave do campo numérico que determina a intensidade de cor</td></tr>
    <tr><td>`color`</td><td>`string`</td><td>`var(--color-extended-blue)`</td><td>Cor base das células (opacidade varia de 0.1 a 1.0 conforme o valor)</td></tr>
    <tr><td>`cell-radius`</td><td>`number`</td><td>`4`</td><td>Raio do arredondamento das células em pixels</td></tr>
    <tr><td>`cell-padding`</td><td>`number`</td><td>`2`</td><td>Espaçamento entre células em pixels</td></tr>
  </tbody>
</table>

### Escala de Opacidade

O heatmap usa uma escala de opacidade linear: o valor mínimo recebe opacidade `0.1` e o máximo recebe `1.0`. Valores intermediários são interpolados. Quando todos os valores são iguais, todas as células recebem opacidade `1.0`.

    <tr><td>`axis-line`</td><td>`boolean`</td><td>`true`</td><td>Exibe a linha do eixo</td></tr>
    <tr><td>`tick-count`</td><td>`number`</td><td>`5`</td><td>Número de ticks no eixo numérico</td></tr>

## Estrutura de Dados

Todos os charts aceitam dados como array de objetos (`ChartDatum[]`) ou string JSON. As chaves padrão são `label` (eixo X) e `value` (eixo Y), mas podem ser customizadas.

```js
// Série única — chaves padrão
const data = [
  { label: 'Jan', value: 100 },
  { label: 'Fev', value: 120 },
  { label: 'Mar', value: 95 },
];

// Multi-série — múltiplos valores por ponto
const multiData = [
  { mes: 'Jan', vendas: 100, meta: 90 },
  { mes: 'Fev', vendas: 120, meta: 110 },
  { mes: 'Mar', vendas: 95,  meta: 100 },
];

// Heatmap — exige x, y e value
const heatData = [
  { dia: 'Seg', hora: '9h',  total: 42 },
  { dia: 'Seg', hora: '10h', total: 78 },
  { dia: 'Ter', hora: '9h',  total: 31 },
];
```
